Actuá como un Product Designer senior + UI Designer editorial premium especializado en landing pages B2B, demos de producto y experiencias de conversión ejecutivas.

Necesito diseñar en Figma una sección / página de demo premium para un servicio de dashboard comercial.
No quiero una estética genérica SaaS ni una maqueta técnica de Power BI.
Quiero una experiencia visual de alto nivel, limpia, sobria, ejecutiva y muy bien resuelta.

OBJETIVO
Diseñar una demo web premium de una solución comercial con 3 vistas:
1. Global
2. Ranking comercial
3. Vendedores

La idea no es mostrar “un dashboard con filtros”.
La idea es mostrar una DEMO GUIADA DE PRODUCTO, como una sola experiencia coherente.

CONTEXTO ESTRATÉGICO
- La vista Global ya funciona bien conceptualmente y debe mantenerse como referencia de calidad.
- El problema actual está más en cómo se siente la experiencia completa y en que las otras vistas no terminan de estar al mismo nivel.
- La demo debe sentirse como una solución comercial premium, orientada a decisiones.
- No debe verse técnica, cortada, improvisada ni “embebida”.
- Tiene que vender criterio comercial, claridad ejecutiva y posibilidad de personalización.

Diseño:
- premium
- editorial
- ejecutivo
- sobrio
- limpio
- con mucho aire
- con excelente jerarquía tipográfica
- con espaciados elegantes
- sin exageración visual
- sin cajas dentro de cajas
- sin look “dashboard crudo”
- sin fondo ruidoso
- sin gradientes estridentes
- sin exceso de sombras

SENSACIÓN BUSCADA
- una sola experiencia integrada
- dashboard protagonista
- texto de apoyo breve y muy bien escrito
- navegación clara
- mucho criterio visual
- foco en negocio, no en tecnología

- gutters generosos
- spacing premium

- lectura ejecutiva integrada

ARQUITECTURA GENERAL
Diseñar una sola sección / página con esta estructura:

1. HEADER DE DEMO
- pequeño eyebrow: DEMO GUIADA
- H1 fijo, que no cambie entre vistas
- subtítulo corto orientado a negocio

El título debe transmitir:
“tablero comercial orientado a decisiones”
No quiero títulos técnicos ni marketineros.
Debe sonar serio, claro y premium.

Ejemplo de intención:
Demo interactiva de tablero comercial
Tres vistas para entender volumen, desempeño y oportunidades de acción

2. NAVEGACIÓN POR VISTAS
Diseñar tabs premium integradas dentro de la misma shell:
- Global
- Ranking comercial
- Vendedores

Las tabs NO deben parecer slicers de Power BI.
Deben parecer navegación de producto.
Estilo:
- pills limpias o segmented control premium
- estado activo elegante
- sin bordes toscos
- con muy buena alineación
- integradas visualmente a la demo

Debajo de las tabs, incluir una micro-descripción dinámica según la vista.
El H1 principal NO cambia.
Solo cambia esta línea contextual.

3. SHELL PRINCIPAL DE DEMO
Diseñar un contenedor principal único, elegante, con:
- tabs
- microcopy contextual
- dashboard grande protagonista
- lectura ejecutiva integrada
- sin sensación de bloque cortado

No quiero dashboard por un lado y texto por otro en cajas separadas.
Quiero una composición coherente y continua.

4. BLOQUE FINAL DE CONVERSIÓN
Debajo de la demo, dentro del mismo flujo visual:
- un cierre breve
- dos CTAs solamente

CTAs:
- Agendar diagnóstico (principal)
- Ver otros indicadores posibles (secundario)

No agregar más acciones.
No repetir CTAs en cada vista.
Todo debe cerrar limpio.

CONTENIDO Y LÓGICA DE CADA VISTA

VISTA 1 — GLOBAL
Esta es la referencia positiva. Debe quedar muy bien.
Es una vista para dirección / liderazgo comercial.

Objetivo:
mostrar salud comercial general sin sobrecargar.

Debe incluir:
- volumen / ventas
- margen o rentabilidad
- segmentación
- una alerta ejecutiva
- tendencia o evolución principal

Estructura sugerida:
- fila superior con 4 KPIs premium
- gráfico principal de tendencia
- bloque de segmentos / mix
- alerta de negocio clara y elegante

La alerta debe ser ejecutiva, no técnica.
Ejemplos de intención:
- caída de margen en un segmento
- concentración de cartera
- desaceleración de una línea
- oportunidad en determinada categoría

No llenar de datos.
Debe verse muy pulida y muy clara.

VISTA 2 — RANKING COMERCIAL
Esta vista evalúa al equipo comercial.
No es para operar clientes, sino para entender performance.

Debe incluir:
- top vendedores vs objetivo
- total cartera por vendedor
- una métrica complementaria que ayude a interpretar el resultado

Elegir la mejor combinación visual entre:
- nuevos clientes
- cartera activa
- ticket promedio
- margen aportado

Pero no recargar.
La idea es que se lea fácil y se entienda:
“no solo importa cuánto vende, sino cómo está construido ese resultado”

Diseñar una composición premium:
- ranking claro
- visual principal fuerte
- información secundaria elegante
- lectura rápida para gerente comercial

Muy importante:
“Total cartera” tiene que estar bien integrado y verse claramente.

VISTA 3 — VENDEDORES
Esta vista NO debe evaluar vendedores.
Debe ser una vista PARA los vendedores.

Concepto:
cartera + foco comercial + stock + oportunidades por cliente

Objetivo:
que un vendedor pueda ver rápido:
- qué clientes priorizar
- qué compra cada cliente
- qué no compra todavía
- dónde hay oportunidad
- si hay alguna restricción de stock

Diseñar esta vista con un enfoque mucho más accionable y comercial.

Estructura sugerida:
A. resumen superior
- clientes activos
- cartera total
- oportunidades detectadas
- alerta de stock

B. bloque principal por cliente
Diseñar una vista premium por cliente.
Puede ser card list editorial, tabla premium o filas enriquecidas.

Cada cliente debería mostrar:
- nombre del cliente
- aporte / volumen / relevancia
- qué categorías o productos compra
- qué categorías o productos no compra
- oportunidad sugerida
- algún indicador visual de cobertura o white space

IMPORTANTE:
Quiero una lectura visual inmediata por cliente.
Podés usar:
- barras horizontales finas
- micrográficos
- coverage bars
- chips por categoría
- indicadores de compra / no compra

La sensación debe ser:
“este cliente me compra A y B, pero no C ni D, ahí tengo una oportunidad”

La alerta de stock debe integrarse como una señal accionable, no como un warning aislado.
Debe afectar visualmente la priorización o sugerencia de acción.

LECTURA EJECUTIVA
Cada vista debe incluir una lectura ejecutiva breve, premium y nada verbosa.
Puede ser un pequeño bloque lateral o inferior con:
- 2 o 3 bullets máximo
o
- 1 mini párrafo muy sintético

El tono debe ser:
- ejecutivo
- claro
- comercial
- útil
- orientado a acción

No lorem ipsum.
Quiero microcopy realista.
